# Zenn AI News Project Rules

## 📋 プロジェクト概要
海外のAI関連ニュースを自動的に収集・要約し、Zennに投稿するシステム

---

## 🎯 記事選定ルール

### 基本方針
- **注目度が高い記事を優先的に選定**
- 技術的価値と実用性のバランスを重視
- 日本の読者にとって有益な情報を提供

### 選定基準（詳細は順次追加）

#### 1. ソース優先度
- [ ] 主要AIニュースサイト（定義予定）
- [ ] 技術ブログ・公式発表
- [ ] 学術論文・研究発表
- [ ] SNSでのバイラル記事

#### 2. 注目度指標
- [ ] ソーシャルメディアでのシェア数
- [ ] コメント数・エンゲージメント
- [ ] 著者・発信元の信頼性
- [ ] 記事の新鮮度（公開日）

#### 3. コンテンツ品質
- [ ] 技術的深さ
- [ ] 実用性・応用可能性
- [ ] 独自性・新規性
- [ ] 分かりやすさ

#### 4. 除外基準
- [ ] 広告・宣伝目的の記事
- [ ] 信頼性の低い情報源
- [ ] 過度に専門的すぎる内容
- [ ] 日本語読者に関連性が低い話題

## 📝 記事作成ルール

### Zenn記事フォーマット
```markdown
---
title: "【AI News】[記事タイトル]"
emoji: "🤖"
type: "tech"
topics: ["AI", "機械学習", "要約"]
published: false  # 必ずfalseで作成
---

# 元記事情報
- **タイトル**: [原題]
- **ソース**: [URL]
- **公開日**: [日付]

# 要約
[3-5段落の要約]

## 重要ポイント
- ポイント1
- ポイント2
- ポイント3

## 技術的詳細
[技術的な説明]

## まとめ
[総括]
```

### 記事作成時の必須ルール
1. **必ず `published: false` で作成**
2. 元記事のURL・タイトルを明記
3. 著作権・引用ルールを遵守
4. 日本語訳は正確かつ分かりやすく
5. 技術用語は適切に解説

## 🛠️ 技術仕様

### 使用ツール
- **Zenn CLI**: 記事管理
- **GitHub**: バージョン管理・自動デプロイ
- **Claude API**: 要約生成
- **WebSearch/WebFetch**: ニュース収集

### ファイル構成
```
zenn-ai-news-project/
├── articles/           # Zenn記事
├── scripts/           # 自動化スクリプト（作成予定）
│   ├── collect_news.py
│   ├── summarize.py
│   └── publish.py
├── config/            # 設定ファイル（作成予定）
│   └── sources.json
└── ZENN_AI_NEWS_RULES.md  # 本ファイル
```
